    Tokens, Embeddings, and Positional Encoding

    How is human language converted into mathematical vectors and how are word positions preserved?

    Transformer Block Map — Current Position

    Focus of this lesson Rest (other lessons)

    Current Position: this lesson covers the entry stage — Input Tokens, Token Embedding and Positional Encoding (highlighted below). The inner Transformer Block comes in later lessons.

    Introduction

    In the last lesson, we saw that AI is just a powerful next-word guesser. But there's a catch: computers can't read words. They only understand numbers!

    Before a sentence can enter the Transformer's brain, it goes through 3 quick steps:

    1. Tokenization: Chopping the sentence into pieces and giving each piece a unique ID number.
    2. Embedding: Turning those numbers into a rich profile (a vector) so the computer understands what the word actually means.
    3. Positional Encoding: Tagging the word with a position number, so the computer knows the order of the sentence.

    Let's explore these concepts simply.

    1. Tokenization (Chopping Text)

    Tokenization is like slicing a pizza. It cuts the input text into small pieces called Tokens. Then, it checks its massive dictionary and assigns an ID number to each piece.

    • Vocabulary: The model's dictionary. For example, GPT-4 has around 100,000 pieces in its dictionary.
    • Token IDs: Once tokenized, the word "cat" is mapped to a unique ID number, such as 3797.
    How does it handle unknown words? (Sub-word Tokenization)

    Instead of memorizing every word in existence, modern AI uses a trick like Byte Pair Encoding (BPE). It keeps common words whole, but chops rare words into familiar chunks.

    For example, if the model doesn't know the word "unhappiness", it splits it into 3 familiar sub-words:
    "un" (ID: 421) + "happi" (ID: 8932) + "ness" (ID: 1205)

    2. Embeddings (Adding Meaning)

    Now we have ID numbers like 3797 for "cat" and 4251 for "dog". But these are just random barcodes. The computer doesn't know that cats and dogs are animals.

    To fix this, we use Embeddings. We replace the simple barcode with a long list of numbers (a Vector). Think of this vector as a detailed "identity profile" (or characteristic card) for the word.

    In this profile, numbers score the word on different traits: Is it an animal? Is it royal? Is it masculine? Words with similar profiles are grouped close together in the Embedding Space.

    Because they are just lists of numbers, we can actually do math with words! For example:

    Vector("King") - Vector("Man") + Vector("Woman") ≈ Vector("Queen")

    3. Positional Encoding (Adding Order)

    The Transformer model is incredibly fast because it reads all the words in a sentence at the exact same time, rather than reading left-to-right.

    But this creates a huge problem: because it reads everything at once, it scrambles the order of the words! Without help, the model can't tell the difference between:

    1. "The cat ate the mouse"
    2. "The mouse ate the cat"

    They contain the exact same words, so they look identical.

    The Solution: Positional Vectors

    To fix this, the model creates a special "position tag" (a Positional Encoding Vector) for word #1, word #2, and so on using math waves (sine and cosine). It mixes this position tag with the word's meaning tag.

    Final Word Profile = Word Meaning (Embedding) + Position Tag
