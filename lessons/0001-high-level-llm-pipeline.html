    Lesson 1

    LLM Pipeline-এর High-Level Intuition

    কীভাবে একটি sentence মডেলের ভেতরে প্রসেস হয়ে output তৈরি করে?

    Transformer Block ম্যাপ — বর্তমান অবস্থান

    সম্পূর্ণ ম্যাপ: নিচের ডায়াগ্রামটি একটি Transformer Block-এর পূর্ণ গঠন দেখায় — নিচে Input Tokens থেকে শুরু করে উপরে Next Token প্রেডিকশন পর্যন্ত। পরের প্রতিটি লেসনে আমরা এর একেকটি অংশ আলাদাভাবে আলোকিত করে গভীরভাবে শিখব।

    The Goal of an LLM

    আপনার ফোনের অটোকমপ্লিট ফিচারের কথা ভাবুন, তবে এটি আরও অনেক স্মার্ট! ChatGPT, Gemini বা Claude-এর মতো Large Language Models (LLMs) মূলত অত্যন্ত শক্তিশালী Next Word Predictors।

    এদের কাজ খুব সহজ: আপনি কিছু টেক্সট (যাকে Prompt বলা হয়) দেবেন, আর মডেল সবচেয়ে সম্ভাব্য পরবর্তী শব্দটি অনুমান করবে। মডেল বারবার এই কাজটি করে এবং নিজের তৈরি করা শব্দগুলোকেই আবার ইনপুট হিসেবে নেয়। এই প্রক্রিয়াটিকে বলা হয় Autoregressive Generation।

    এই "ম্যাজিক" যার ওপর দাঁড়িয়ে আছে, তা হলো Transformer Architecture।

    The Core Pipeline (৪টি মূল ধাপ)

    মডেলটিকে একটি ফ্যাক্টরির অ্যাসেম্বলি লাইনের মতো কল্পনা করুন। আপনি একটি Prompt টাইপ করার পর মূল ৪টি ধাপ নিচে দেওয়া হলো:

    1

    Tokenization (টেক্সট টুকরো করা)

    কম্পিউটার শব্দ বোঝে না, শুধু সংখ্যা বোঝে। তাই ইনপুট টেক্সটকে ছোট ছোট টুকরোতে ভাগ করা হয়, যাদের Tokens বলে। এটি অনেকটা পিৎজা স্লাইস করার মতো। প্রতিটি টুকরোকে একটি ইউনিক ID নম্বর দেওয়া হয়।

    2

    Embedding (অর্থ যোগ করা)

    এবার প্রতিটি ID-কে একটি সংখ্যার তালিকায় (যাকে Vector বলা হয়) রূপান্তর করা হয়। এটি শব্দের একটি প্রোফাইলের মতো—যা শব্দের বৈশিষ্ট্য এবং অর্থ বর্ণনা করে, যাতে কম্পিউটার বুঝতে পারে যে "রাজা" এবং "রাণী" সম্পর্কিত শব্দ।

    3

    Transformer Blocks (মডেলের মস্তিষ্ক)

    এখানেই আসল ম্যাজিক ঘটে। Self-Attention Mechanism ব্যবহার করে মডেলটি বাক্যের কনটেক্সট বা প্রসঙ্গ বোঝার জন্য একই সাথে সব শব্দের দিকে তাকায়। এটি অনেকটা পুরো বাক্য পড়ে বোঝার মতো যে, "তীর" বলতে নদীর তীর বোঝানো হচ্ছে নাকি ধনুকের তীর।

    টেকনিক্যাল বিষয়গুলো জানতে চান?

    এটি প্রতিটি টোকেনকে তার আশেপাশের অন্যান্য টোকেনগুলোর দিকে তাকাতে সাহায্য করে এবং Query, Key, এবং Value ম্যাট্রিক্স ব্যবহার করে বুঝতে পারে কোন শব্দের সাথে কোন শব্দ সম্পর্কিত (Context Extraction)।

    4

    Output & Softmax (অনুমান করা)

    অবশেষে, মডেল তার সম্পূর্ণ ডিকশনারি বা ভোকাবুলারির প্রতিটি শব্দের জন্য একটি স্কোর বের করে। Softmax ফাংশন এই স্কোরগুলোকে শতকরা হার (percentage)-এ রূপান্তর করে এবং সর্বোচ্চ শতকরা হারের শব্দটি বিজয়ী হয়!

    Interactive Pipeline Simulator

    চলুন ফ্যাক্টরিটিকে কাজ করতে দেখি! নিচের যেকোনো একটি Prompt সিলেক্ট করে "Run Step-by-Step" বাটনে ক্লিক করুন এবং দেখুন কীভাবে শব্দগুলো ৪টি ধাপের মধ্যে দিয়ে প্রবাহিত হচ্ছে।

    টেক্সটকে টোকেনে বিভক্ত করা হয়েছে এবং প্রতিটির জন্য Vocabulary ID নির্ধারণ করা হয়েছে:

    Prompt সিলেক্ট করে Run চাপুন...

    প্রতিটি টোকেন এখন একটি 8-dimensional Vector (বাস্তবে এটি ৪০৯৬+ ডাইমেনশন হয়):

    Prompt সিলেক্ট করে Run চাপুন...

    Self-Attention-এর মাধ্যমে শেষ টোকেনটি পেছনের টোকেনগুলোর ওপর কতখানি মনোযোগ (Attention Weight) দিচ্ছে তা দেখুন:

    Prompt সিলেক্ট করে Run চাপুন...

    Quick Assessment (কুইজ)

    আপনার ধারণাকে ঝালাই করতে নিচের প্রশ্নটির উত্তর দিন:

    প্রশ্ন: Transformer block-এর ভেতরে থাকা Self-Attention Mechanism-এর মূল কাজ কী?

    ক) টেক্সটকে টুকরো টুকরো করে টোকেনে ভাগ করা।
    খ) একটি বাক্যের বিভিন্ন টোকেনের মধ্যকার পারস্পরিক সম্পর্ক এবং কনটেক্সট তৈরি করা।
    গ) পরবর্তী টোকেনের সম্ভাবনা বা Probability স্কোর গণনা করা।
